## Summary

This project implements a complete **Adaptive Traffic Signal Control (ATSC)** framework combining four main contributions:

1. **Offline pre-training** with a *Decision Transformer* (DT) on a historical corpus generated by baseline controllers — solves the sample inefficiency of classical RL.
2. **Formal safety shield** via *Action Masking* — guarantees **zero regulatory violations** by design, regardless of the agent policy.
3. **LemgoRL benchmark** with a realistic 4-intersection urban corridor topology — maximises sim-to-real transferability.
4. **Corridor coordination** with green-wave synchronisation between traffic lights — reduces travel times at the full corridor level.

### Main result

> **DT with online fine-tuning** outperforms the fixed-time controller by **+37.3% reward**, **-34.8% travel time** and **+29.5% throughput**, maintaining **0 safety violations** across all episodes.

## 3-Phase Pipeline

```
+------------------------------------------------------------------+
|  PHASE I: Offline Pre-training                                   |
|                                                                  |
|  SUMO + LemgoRL --> FixedTime/Actuated --> Dataset (1000 ep)     |
|                                   |                              |
|                                   v                              |
|                        Decision Transformer                      |
|                       (offline pre-training)                     |
|                        1,271,696 parameters                      |
+-----------------------------+------------------------------------+
                              | best_model.pt
                              v
+------------------------------------------------------------------+
|  PHASE II: Safety + Coordination                                 |
|                                                                  |
|  Action Masking --> min_green=5s, intergreen=3s, max_red=120s    |
|  Corridor Coordinator --> Green Wave offsets [0,14,28,43]s       |
+-----------------------------+------------------------------------+
                              | model + shield
                              v
+------------------------------------------------------------------+
|  PHASE III: Online Fine-tuning + Benchmarking                    |
|                                                                  |
|  200 online episodes --> FIFO replay --> last 2 layers           |
|  Benchmark: Fixed-time vs Actuated vs DT-offline vs DT-online    |
+------------------------------------------------------------------+
```

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/RaulVelasquez/Safe-Offline-RL.git
cd Safe-Offline-RL
pip install -e .
```

### 2. Install PyTorch with CUDA (recommended)

```bash
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124
```

### 3. Install SUMO

**Windows:**
```bat
setup_windows.bat
```

**Linux/macOS:**
```bash
sudo apt-get install sumo sumo-tools
export SUMO_HOME=/usr/share/sumo
bash scripts/setup_lemgorl.sh
```

### 4. Verify the environment

```bash
python main.py --check
```

## Usage

```bash
# Full pipeline (Phases I -> II -> III)
python main.py --phase all

# Individual phase
python main.py --phase 1    # Offline dataset + DT pre-training
python main.py --phase 2    # Action Masking validation + green wave
python main.py --phase 3    # Online fine-tuning + benchmarking

# Synthetic simulation + figures (no SUMO required)
python main.py --phase sim
```

### Resume Phase III from last checkpoint

```bash
# Automatically detects the last ep_XXXX.pt in checkpoints/online/
python main.py --phase 3
```

### Generate paper figures only

```bash
cd simulation
python run_simulation.py   # Generates CSVs (~1 sec)
python plot_results.py     # Generates 8 figures (~10 sec)
```

## Decision Transformer — Architecture

```python
model = DecisionTransformer(
    obs_dim        = 100,   # 4 intersections x 25 features
    act_dim        = 16,    # 4 intersections x 4 phases
    num_tls        = 4,
    context_length = 20,    # context window K
    d_model        = 128,   # internal dimension
    n_layer        = 4,     # transformer blocks
    n_head         = 4,     # attention heads
    d_inner        = 512,   # FFN width
    dropout        = 0.1,
    max_ep_len     = 3600,  # steps per episode
)
# Total parameters: 1,271,696
```

Conditional prediction on Return-to-Go:

```
a_t ~ pi(· | R_t, s_t, a_{t-1}, ..., R_{t-K+1}, s_{t-K+1})
```

## Action Masking — Formal Safety

```python
mask = ActionMask(
    num_tls             = 4,
    num_phases          = 4,
    min_green           = 5,    # minimum seconds in green
    min_intergreen      = 3,    # intergreen seconds
    max_consecutive_red = 120,  # maximum seconds in red
)

safe_actions, violations = mask.update(agent_actions)
# violations == 0 ALWAYS (formal guarantee)
```

## Critical Hyperparameters

| Parameter | Value | Description |
|---|---|---|
| `context_length` | 20 | DT context window K |
| `d_model` | 128 | Transformer internal dimension |
| `n_layer` | 4 | Attention blocks |
| `target_return` | -50.0 | RTG target (conditional RTG) |
| `offline_lr` | 1e-4 | Offline learning rate (AdamW) |
| `online_lr` | 5e-5 | Fine-tuning learning rate |
| `trainable_layers` | 2 | Unfrozen layers in online phase |
| `steps_per_epoch` | 500 | Steps per epoch (CPU/GPU viable) |
| `buffer_capacity` | 10,000 | Online replay buffer (FIFO) |
